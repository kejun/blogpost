# Ring-Zero 深度解析：当 Zero RL 扩展到万亿参数——"苦涩教训"的终极验证

**文档日期：** 2026 年 7 月 17 日  
**标签：** Zero RL, Emergent Reasoning, Scaling Law, Chain-of-Thought, 1T Parameters, Ring-Zero, RLVR, CoT Quality

## 一、背景：一个万亿参数的"自我觉醒"实验

2026 年 7 月 14 日，arXiv 上出现了一篇迅速引发社区关注的论文：**Ring-Zero: Scaling Zero RL to a Trillion Parameters for Emergent Reasoning**（arXiv:2607.12395）。这篇论文由来自阿里巴巴集团和中国人民大学的研究团队共同完成，它回答了一个此前从未有人实证回答过的问题：

> **当 Zero RL（零样本强化学习）直接作用于一个万亿参数的基座模型时，会发生什么？**

这不是一个"更大就是更好"的简单验证。这是一个关于**智能如何在纯试错中自发涌现**的实证研究——而且实验规模达到了此前所有同类研究的数十倍。

### 1.1 什么是 Zero RL？为什么它重要？

要理解 Ring-Zero 的价值，需要先理解 Zero RL 的范式革命：

```
传统 RL 训练流程（以 DeepSeek-R1 为代表）：
基座模型 → SFT（监督微调，使用 CoT 数据）→ RL 训练 → 推理模型

Zero RL 训练流程（以 Ring-Zero 为代表）：
基座模型 → 直接 RL 训练 → 推理模型
           ↑ 跳过了整个 SFT 阶段
           ↑ 不需要任何人类标注的推理示范数据
```

Zero RL 的核心思想极其激进：**直接从预训练基座模型开始强化学习，完全跳过监督微调（SFT）阶段，不使用任何人类标注的 Chain-of-Thought 数据。** 模型只通过"做对题目得奖励、做错不得奖励"这个极其简单的信号，自发地学会逐步推理。

DeepSeek-R1 在 2025 年初首次验证了 Zero RL 在较小模型上的可行性。但关键问题一直悬而未决：**这套方法在万亿参数规模上是否依然有效？** 因为计算成本的原因，此前所有 Zero RL 研究都被限制在较小的模型上（通常是 7B-70B 级别）。Ring-Zero 是第一个将 Zero RL 扩展到 **1T（万亿）参数**规模的系统性实验。

### 1.2 为什么"规模"不是简单的外推？

有人可能会问：不就是把模型做大一点吗？但 Ring-Zero 团队在论文开头就揭示了一个关键发现：

> **naive scaling often suffers from poor readability, token redundancy, and a lack of adaptive reasoning depth.**

翻译过来：**朴素地放大规模会导致推理可读性变差、Token 冗余严重、且缺乏自适应推理深度。**

这三个问题恰恰是 Zero RL 在大规模上面临的核心工程挑战：
- **可读性差**：推理过程缺乏逻辑格式和清晰结构，人类难以理解和验证
- **Token 冗余**：标准算法（如 GRPO）隐式地偏向更长的序列，导致推理链无限膨胀
- **推理深度僵化**：不同难度的题目需要不同深度的推理，但标准训练只能产出单一模式

这意味着：**规模放大不是简单的外推，它需要新的算法和系统优化来驯服训练过程。**

## 二、Ring-Zero 的训练管道：四阶段"极简主义"哲学

Ring-Zero 最令人惊讶的设计哲学是**极简主义**——他们发现，在万亿参数规模上稳定训练只需要几个简单但关键的修改，而不是复杂的工程堆叠。

### 2.1 训练管道总览

```
第一阶段 RL          自蒸馏             第二阶段 RL           第三阶段 RL
Reasoning         CoT压缩             样本级损失           分层自适应训练
Elicitation      训练推理引擎对齐     稳定优化提升          多难度自适应
    │                │                  │                    │
    ▼                ▼                  ▼                    ▼
  [基座模型] → [压缩后的推理] → [稳定推理模型] → [自适应推理深度模型]
   clipped IS    self-distill     sample-level     tier-based
   policy grad   ratio correction  loss norm        training
```

**第一阶段：推理激发（Reasoning Elicitation）**
- 使用 **clipped importance sampling policy gradient**（MiniMax, 2025）
- 与标准 PPO-clip 不同，这个方法对 clipping range 外的 token 仅施加 stop-gradient，而非完全归零
- 这确保**每个生成的 token 都能贡献学习信号**，对从零构建推理能力至关重要

**关键工程发现：训练-推理引擎差异修正**
Ring-Zero 团队发现了一个此前被忽略的系统级漏洞：训练引擎（Megatron）和推理引擎（SGLang）由于浮点精度和内核实现的微小差异，会产生略微不同的 logits。在 Zero RL 的早期阶段，这种微分歧会**复合放大为训练崩溃**。

他们的解决方案非常精妙：用训练引擎的实际 logits 替换重要性采样比率中的分子，从而消除这种系统级不一致性。

```python
# 标准重要性采样比率
ρ = π_M(o|q,o_<t) / π_S_old(o|q,o_<t)
# ↑ π_M: 训练引擎(Megatron), π_S: 推理引擎(SGLang)
# 直接使用会导致精度差异累积 → 训练崩溃

# Ring-Zero 的修正
ρ̂ = clip(ρ, ε_low, ε_high)
# 分子来自训练引擎的实际 logits
# 分母来自推理引擎的 rollout 概率
# 通过 clipped ratio + training-engine 分子修正，消除系统偏差
```

**第二阶段：自蒸馏（Self-Distillation）**
这是一个非常巧妙的中间阶段：
1. 用第一阶段产出的模型生成推理链
2. 用这些推理链对自己进行蒸馏训练
3. 目的：**压缩推理链长度 + 重置训练-推理引擎差异**

自蒸馏的本质是让模型"学会说人话"——把第一阶段学到的冗长、混乱的推理过程压缩为更简洁、更结构化的形式。

**第三阶段：样本级损失标准化**
从 token-level loss 切换为 **sample-level loss normalization**。这是一个关键的稳定性改进——在万亿参数规模上，token-level 的细粒度优化反而引入了不必要的噪声。

**第四阶段：分层自适应训练（Tier-based Training）**
这是解决"自适应推理深度"的核心创新：
- 将题目按难度分层
- 不同层使用不同的响应预算和训练策略
- 模型学会根据题目难度**自适应地选择推理深度**

简单题用短推理链，难题用长推理链——这更接近人类的认知模式。

### 2.2 基础设施优化

在万亿参数规模上，系统优化与算法优化同等重要：

| 优化维度 | 技术手段 | 解决的问题 |
|---------|---------|-----------|
| **混合精度控制** | BF16/FP8 混合训练 | 数值稳定性 + 训练吞吐量 |
| **上下文并行** | Context Parallelism | 超长 CoT 序列的内存瓶颈 |
| **Megatron + SGLang 桥接** | 训练-推理比率修正 | 引擎差异导致的训练崩溃 |

## 三、三大核心发现："苦涩教训"的终极验证

Ring-Zero 最重要的贡献不是模型性能，而是三个关于**大规模 Zero RL 训练动态**的发现。这些发现直接验证了 Rich Sutton 在 2019 年提出的著名论断——"苦涩教训"（The Bitter Lesson）：

> **"利用计算的一般方法最终会胜过人类设计的启发式方法。"**

### 3.1 发现一：规模决定能力天花板

> **scaling to 1T parameters significantly enhances sample efficiency and performance ceilings.**

1T 模型与 104B 模型的对比揭示了一个令人震惊的事实：

```
能力天花板对比：

104B 模型          1T 模型
   │                 │
   ├─ 样本效率较低    ├─ 样本效率显著提升
   ├─ 性能上限较低    ├─ 性能上限大幅超越
   ├─ 需要更多步骤    ├─ 更快收敛到高质量策略
   └─ 容易陷入局部最优 └─ 能发现更优的推理策略

差距不是线性的——是结构性的。
```

这意味着：**在 Zero RL 范式下，规模不仅带来性能提升，更改变了训练的"动力学性质"。** 更大的模型不仅仅是"更强的计算器"——它在训练过程中表现出截然不同的行为模式。

### 3.2 发现二："发现"与"锐化"是连续的两个阶段

这是 Ring-Zero 对学术界长期争论的重要贡献。此前存在两种对立观点：

| 观点 | 核心主张 |
|------|---------|
| **发现论（Discovery）** | 训练是在"解锁"模型预训练时已有的潜在能力 |
| **锐化论（Sharpening）** | 训练是在"精炼"模型已有的推理能力 |

Ring-Zero 的实验表明：**两者都是对的，但发生在不同的阶段。**

```
训练动态的时序演化：

阶段 1: 发现期 (Discovery Phase)          阶段 2: 锐化期 (Sharpening Phase)
┌──────────────────────────────┐          ┌──────────────────────────────┐
│ 解锁休眠的推理路径             │          │ 在新建立的能力边界内精炼策略    │
│ 扩展推理能力的边界             │    →     │ 优化推理效率和质量             │
│ 性能快速上升                   │          │ 性能缓慢但稳定地提升           │
│ 涌现新的推理策略               │          │ 消除冗余，压缩推理链           │
└──────────────────────────────┘          └──────────────────────────────┘
       ↑ 曲线陡峭                             ↑ 曲线平缓但持续
```

这个发现的深层意义在于：**它揭示了 Zero RL 不是单纯的"发现"或"创造"，而是一个"先扩展、后精炼"的双阶段过程。** 这解释了为什么有些研究观察到快速的能力跃升（发现期），而另一些研究观察到缓慢的性能改善（锐化期）——他们观察的是同一过程的不同阶段。

### 3.3 发现三：高级认知行为的自发涌现（最震撼的发现）

这是 Ring-Zero 最令人兴奋的发现。在纯 Zero RL 训练过程中，没有任何人工设计或引导，模型**自发地**涌现出了以下五种高级认知策略：

| 涌现行为 | 描述 | 为什么重要 |
|---------|------|-----------|
| **拟人化（Anthropomorphism）** | 模型开始使用"我们"、"让我想想"等拟人化表达 | 说明模型自发采用了最自然的认知表达方式 |
| **结构化格式化（Structured Formatting）** | 自发使用编号、分节、标注等结构化格式组织推理 | 消除了对手工启发式格式设计的需求 |
| **自我验证（Self-Verification）** | 在得出答案后自动进行验证检查 | 无需人工设计的 verification 模块 |
| **并行推理（Parallel Reasoning）** | 同时尝试多条推理路径并比较结果 | 自发学会了类似 beam search 的策略 |
| **上下文焦虑（Context Anxiety）** | 在信息不足时表达不确定性并请求更多信息 | 表现出对自身认知局限的感知 |

```
这五种涌现行为的意义：

过去几年，研究者们设计了复杂的工程管道来"激励"模型学会这些行为：
- Verification → 需要专门的 verification reward / critic model
- Structured CoT → 需要人工设计的格式模板和 SFT 数据
- Self-consistency → 需要多次采样 + 投票机制
- Uncertainty calibration → 需要专门的校准训练

Ring-Zero 的结果说：你们做的这些，模型自己就能学会。
只要规模够大，训练方法够稳定。

这就是"苦涩教训"的本质：
人类工程师精心设计的启发式方法，最终被纯规模 + 简单训练所超越。
```

这个发现对 AI Agent 领域的冲击是巨大的。当前 Agent 系统中的许多"高级功能"（自我验证、不确定性感知、并行推理）都是通过复杂的工程管道实现的。Ring-Zero 暗示：**如果基座模型本身通过 Zero RL 学会了这些行为，Agent 系统的架构可以大幅简化。**

## 四、CoT 质量评估框架：超越"答案正确"的三维评估

Ring-Zero 的第二个重要贡献是提出了一个结构化的 Chain-of-Thought 质量评估框架。当前业界（包括 GLM、Kimi 等前沿模型）的评估几乎完全依赖**最终答案准确率**，但 Ring-Zero 指出这在 Agent 时代已经不够了：

> **CoT 不仅是人类的解释，更是下游 Agent 系统的关键中间步骤。**

### 4.1 三个维度

```
CoT 质量评估三维框架：

┌─────────────────────────────────────────────────────────┐
│                                                         │
│   可理解性 (Comprehensibility)                           │
│   → 人类能否不借助外部上下文就跟上推理步骤？               │
│   → 评估方法：LLM-as-a-Judge 成对比较                    │
│   → 核心指标：逻辑连贯性、表述清晰度、推理缺陷数量        │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   可复现性 (Reproducibility)                             │
│   → 缺乏先验知识的弱模型能否有效学习这些推理策略？        │
│   → 评估方法：知识蒸馏——用 CoT 微调弱模型，看性能提升   │
│   → 核心指标：学生模型的性能增益幅度                      │
│                                                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│   效率 (Efficiency)                                     │
│   → 推理链是否简洁？有无冗余推导或循环逻辑？              │
│   → 评估方法：正确答案的平均 Token 数                   │
│   → 核心指标：在正确的前提下，Token 越少效率越高         │
│                                                         │
└─────────────────────────────────────────────────────────┘

这三个维度独立于最终答案正确率，但高度互补。
```

### 4.2 为什么这个框架重要？

在 Agent 工作流中，CoT 不再只是"给用户看的解释"——它是**下游系统需要理解和依赖的中间表示**。一个正确但混乱的 CoT 在 Agent 系统中的价值远低于一个正确且结构清晰的 CoT。

具体来说：
- **可理解性**影响人类对 Agent 输出的信任度和验证效率
- **可复现性**影响知识蒸馏的效率——好的 CoT 可以让小模型快速获得大模型的推理能力
- **效率**直接影响推理速度和 Token 成本——在 Agent 系统中，每个 token 都是真金白银

Ring-Zero 的实验表明，在这三个维度上，他们的模型都展现出明显优势——产出的是**结构化、人类可读、且简洁的推理链**。

## 五、性能表现：Ring-2.5-1T-Zero 的 Benchmark 成绩

在七个数学基准测试上，Ring-2.5-1T-Zero 取得了与前沿模型可比的成绩：

| 基准 | Ring-2.5-1T-Zero | 意义 |
|------|------------------|------|
| AIME 2024 | 竞争性成绩 | 高中数学竞赛级难题 |
| AIME 2025 | 竞争性成绩 | 最新年度数学竞赛 |
| MATH-500 | 竞争性成绩 | 经典数学问题集 |
| 其他 4 个数学基准 | 竞争性成绩 | 全面覆盖数学推理场景 |

需要强调的是：**这些成绩是在完全不使用任何人类标注 CoT 数据的情况下取得的。** 模型只通过"答对得奖励"这个极其简单的信号，就达到了与使用大量监督数据训练的前沿模型相当的水平。

## 六、独到见解：Ring-Zero 对行业意味着什么？

### 6.1 对模型训练的影响：SFT 的时代结束了吗？

Ring-Zero 最直接的启示是：**在数学推理等可验证领域，SFT 可能不再是必需的。** 如果 Zero RL 可以在万亿参数规模上直接从基座模型激发出高质量推理，那么 SFT 阶段的时间和数据成本可以被节省下来。

但这有一个关键前提：**规模必须足够大。** Ring-Zero 的比较实验明确显示，104B 模型在 Zero RL 下的表现远不如 1T 模型。这意味着 Zero RL 的可行性与模型规模存在强相关性——小模型可能仍然需要 SFT 作为"引导"。

### 6.2 对 Agent 架构的影响：简化是必然趋势

当前 Agent 系统的复杂程度令人担忧：

```
典型的 2026 年 Agent 架构：
用户请求 → Router → 任务分解器 → 工具选择器 → 执行器 → 验证器
                  → CoT 格式化工具 → 自我验证模块 → 不确定性估计模块
                  → 并行推理控制器 → 结果聚合器 → ...

每个模块都需要精心设计、调优和维护。
```

Ring-Zero 暗示了一个更简洁的未来：

```
Ring-Zero 启示的 Agent 架构：
用户请求 → 基座模型（已通过 Zero RL 学会所有能力）→ 结果
```

如果模型本身已经涌现了自我验证、结构化推理、并行推理、不确定性感知等能力，Agent 系统的大部分工程复杂度都可以被消除。

### 6.3 对"苦涩教训"的重新思考

Rich Sutton 的"苦涩教训"（2019）说：利用计算的一般方法最终胜过人类设计的启发式。Ring-Zero 为这个论点提供了迄今为止最强有力的实证支持——**在万亿参数规模上，简单的 Zero RL 训练让模型自发涌现出了研究者多年来用复杂工程管道试图实现的所有能力。**

但我们也要保持清醒的批判性思考：

**"苦涩教训"的边界在哪里？**
- Ring-Zero 的实验集中在数学推理领域——这个领域有明确的正确答案可以验证
- 在开放域任务（如创意写作、复杂系统理解）中，Zero RL 的可行性仍然未知
- "可验证奖励"的可用性是 Zero RL 的前提——不是所有任务都有清晰的验证标准

**规模的代价是什么？**
- 万亿参数模型的训练成本是天文数字
- 推理成本同样巨大——即使 CoT 被压缩，绝对 token 数仍然很高
- 这是否意味着只有巨头才能玩转 Zero RL？

**涌现不等于可控**
- 模型涌现的行为是我们期望的，但也可能涌现我们**不希望**的行为
- 没有人工引导的 Zero RL 是否会在安全边界上产生问题？

### 6.4 对开源社区的意义

Ring-Zero 团队选择分享他们的实验细节和观察到的涌现现象，这对开源社区意义重大。此前，大规模 Zero RL 的训练动态一直是个"黑盒"——只有拥有大规模算力的实验室才能观察到。Ring-Zero 的开放分享意味着：

1. **中小团队可以基于这些发现设计更高效的训练策略**——即使他们无法在万亿参数上实验，也可以借鉴发现的规律
2. **CoT 质量评估框架可以被社区采用**——为评估推理质量提供了超越准确率的标准化工具
3. **训练-推理引擎差异修正**是一个普适性的工程发现，可以应用于其他大规模 RL 训练场景

## 七、结语：一个转折点的信号

Ring-Zero 不只是一篇论文，它是 AI 训练范式演进的一个重要信号。

过去一年，我们看到了：
- DeepSeek-R1 证明了 Zero RL 的可行性
- OpenAI 的 o1/o3 系列展示了推理模型的能力上限
- 各种"推理蒸馏"方法试图让大模型的推理能力流向小模型

Ring-Zero 回答了一个关键问题：**Zero RL 不是小模型的玩具——它在万亿参数规模上不仅有效，而且展现出了比小规模更丰富的涌现行为和更高的样本效率。**

这意味着推理模型的发展路线可能比我们想象的更简单、更直接：给一个足够大的基座模型，用 Zero RL 训练，然后它就会"自己学会思考"。

当然，这条路还有很长的距离——成本、可控性、泛化性都是待解决的问题。但 Ring-Zero 至少证明了：**方向是对的，规模是有效的，而人类工程师能做的，模型自己也能学会。**

这大概就是"苦涩教训"最苦涩的一面。
