# Subquadratic 稀疏注意力突破：终结 Transformer 二次方瓶颈，还是 AI 版 Theranos？

> **摘要：** 2026 年 6 月，迈阿密 AI 初创公司 Subquadratic 从隐身模式走出，宣称其 SubQ 模型解决了困扰 LLM 近十年的注意力机制二次方计算瓶颈。第三方评测机构 Appen 的独立验证显示 SubQ 在编码和长上下文任务上确实达到了前沿水平，且速度是 FlashAttention 的 56 倍、RULER 128 测试成本从 $2600 骤降至 $8。但学术界仍然怀疑：动态稀疏选择机制究竟如何工作？模型权重复用 Qwen 是否削弱了其"全新架构"的宣称？本文将从 Transformer 注意力机制的第一性原理出发，深度拆解 Subquadratic 的技术路径，对比历史上所有稀疏注意力尝试的失败教训，并回答一个更根本的问题：**如果稀疏注意力真的可行，为什么 Google、OpenAI、Anthropic 没有率先做到？**

---

## 一、背景：二次方注意力的十年枷锁

### 1.1 "Attention Is All You Need" 的计算代价

2017 年 Vaswani 等人的论文 *"Attention Is All You Need"* 奠定了 Transformer 架构的基础。这篇论文标题中的"All You Need"在 2026 年看来，可能恰恰是问题所在。

**密集注意力（Dense Attention）的计算复杂度是 O(n²)**。这个 n² 来自哪里？

```
┌─────────────────────────────────────────────────────────────────┐
│  密集注意力的计算过程                                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  输入：n 个 tokens，每个 token 映射为 d 维向量                     │
│                                                                  │
│  步骤 1：计算 QK^T                                              │
│    Q: [n × d]  K: [n × d]                                       │
│    QK^T: [n × d] × [d × n] = [n × n]  ← 这里就是 n²             │
│                                                                  │
│  步骤 2：对每一行做 softmax                                       │
│    softmax([n × n]) → [n × n]  注意力权重矩阵                     │
│                                                                  │
│  步骤 3：用注意力权重加权 Value                                   │
│    [n × n] × [n × d] = [n × d]                                  │
│                                                                  │
│  总计算量：O(n² × d)                                             │
│  总内存：O(n²) 存储注意力权重矩阵                                 │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

用一个具象的例子：**《了不起的盖茨比》全书约 47,000 个词。** 如果用密集注意力处理这本书，需要计算 47,000 × 47,000 ≈ **22 亿次** token 对之间的注意力。即使使用 FlashAttention 这样的 I/O 感知优化算法，这个二次方关系也无法消除。

### 1.2 二次方扩展的实际影响

让我们量化一下这个瓶颈到底有多痛：

| 上下文长度 | 注意力矩阵大小 | 相对计算量（以 1K 为基准） |
|------------|---------------|--------------------------|
| 1K tokens | 1M 元素 | 1× |
| 128K tokens | 16.4G 元素 | 16,384× |
| 1M tokens | 1T 元素 | 1,000,000× |
| 12M tokens（SubQ 宣称） | 144T 元素 | 144,000,000× |

**上下文长度翻倍，计算量变为四倍。** 这就是为什么当前主流模型的上下文窗口大多卡在 128K–1M tokens——不是不想更长，而是密集注意力的二次方复杂度让更长的上下文变得经济上不可行。

### 1.3 历史上的稀疏注意力尝试

Subquadratic 并非第一个尝试打破二次方瓶颈的团队。事实上，过去十年中几乎所有主要的研究机构和公司都做过尝试：

| 方法 | 提出者 | 年份 | 核心思路 | 为什么失败 |
|------|--------|------|----------|------------|
| **Sparse Transformer** | OpenAI (Child et al.) | 2019 | 固定模式的稀疏注意力（strided + fixed） | 固定模式无法捕捉远距离语义依赖，质量显著下降 |
| **Longformer** | Allen AI (Beltagy et al.) | 2020 | 滑动窗口 + 全局 tokens | 滑动窗口大小是超参数，无法自适应不同任务 |
| **BigBird** | Google (Zaheer et al.) | 2020 | 随机 + 滑动窗口 + 全局 tokens | 理论上是 universal approximator，但实践中随机连接质量不稳定 |
| **Performer** | Google (Choromanski et al.) | 2021 | FAVOR+ 核方法近似 softmax 注意力 | 近似误差在长序列中累积，语言建模质量不及标准注意力 |
| **Linear Attention** | 多个团队 | 2020-2023 | 将 softmax 替换为线性核函数 | 语言建模能力系统性下降，无法通过缩放定律补偿 |
| **FlashAttention** | Stanford/Dao et al. | 2022-2023 | I/O 感知的精确注意力，不改变算法复杂度 | 只优化了常数因子和内存带宽，复杂度仍是 O(n²) |

**核心教训：所有"降低复杂度"的方法都付出了质量代价。** 在短上下文上不明显，但在长上下文和复杂推理任务中，质量差距会迅速放大。这就是为什么尽管学术界提出了数十种替代方案，所有前沿模型仍然坚持使用密集注意力。

Subquadratic 的宣称之所以引起轰动（和怀疑），是因为它声称**在不牺牲质量的前提下，打破了这个 trade-off**。

## 二、Subquadratic 的技术路径：动态稀疏注意力

### 2.1 核心理念：不是所有 token 对都需要互相看

Subquadratic 的 CTO Alex Whedon 用一个直观的比喻解释了他们的动机：

> *"如果你在读一本书，你不会去看第一个词和第二个词、第一个词和第三个词的关系——那太荒谬了。"*

这个直觉在认知科学上是有根据的。人类阅读时确实只关注与当前理解相关的部分文本。问题在于：**让模型在运行时自己判断"哪些 token 对是重要的"，比用固定模式选择要难得多。**

### 2.2 从固定稀疏到动态稀疏

历史上失败的方法大多使用了**固定的稀疏模式**：

```
┌──────────────────────────────────────────────────────────────┐
│  固定稀疏模式 vs 动态稀疏选择                                │
├──────────────────────────────────────────────────────────────┤
│                                                               │
│  固定稀疏（Longformer / Sparse Transformer）：                 │
│  ┌─┬─┬─┬─┬─┬─┬─┬─┐                                          │
│  │█│█│░│░│░│░│░│░│  token 0 只看邻近 2 个 tokens               │
│  │░│█│█│█│░│░│░│░│  token 1 只看邻近 2 个 tokens               │
│  │░│░│█│█│█│░│░│░│  所有 token 遵循同一规则                    │
│  │░│░│░│█│█│█│░│░│  → 无法适应不同语义结构                     │
│  └─┴─┴─┴─┴─┴─┴─┴─┘                                          │
│                                                               │
│  动态稀疏（SubQ 宣称）：                                       │
│  ┌─┬─┬─┬─┬─┬─┬─┬─┐                                          │
│  │█│░│░│█│░│░│░│░│  token 0 根据内容选择看哪些 tokens           │
│  │░│█│░│░│░│█│░│░│  token 1 的选择不一样                       │
│  │█│░│█│░│░│░│░│░│  token 2 又不一样                           │
│  │░│░│░│█│█│░│█│░│  → 选择随输入内容动态变化                    │
│  └─┴─┴─┴─┴─┴─┴─┴─┘                                          │
│                                                               │
└──────────────────────────────────────────────────────────────┘
```

**动态稀疏的理论难点在于：** 你需要用一种计算量远小于 O(n²) 的方法，来决定哪些 token 对应该被关注。如果你用来做选择的方法本身就需要 O(n²)，那就失去了意义。

### 2.3 SubQ 的技术细节（基于公开信息）

Subquadratic 尚未开源 SubQ 模型，也未发表同行评审论文。我们只能通过其公开的 model card、MIT Technology Review 的采访、以及 Appen 的第三方评测来拼凑技术图景：

**已知信息：**

1. **基于 Qwen 权重引导训练：** SubQ 复用了某个版本的 Qwen 开源模型的权重来"bootstrap"训练，而非从头开始。这在开源社区是常见做法（如 Llama 系列基于公开预训练权重微调），但削弱了 Subquadratic"完全重新发明 LLM"的宣称。

2. **动态稀疏选择机制：** 据 CTO Whedon 的描述，"选择在运行时计算，对每段输入的文本都不同"。具体算法被描述为"secret sauce"，未公开。

3. **上下文窗口：** 最大 12M tokens（1200 万），是当前主流模型（128K–1M）的 12–100 倍。

4. **专注领域：** 编码（coding）和大规模数据集检索（large-scale data retrieval）。

### 2.4 与 FlashAttention 的关系

一个重要的技术细节：Subquadratic 声称 SubQ 的速度是 **FlashAttention 的 56 倍**。这说明他们的比较基线不是原始的密集注意力，而是当前最先进的精确注意力实现 FlashAttention。

这一点很重要。FlashAttention 已经将密集注意力的实际运行速度提升了 2-4 倍（通过优化 GPU 内存访问模式），但它没有改变算法的渐近复杂度。SubQ 如果能比 FlashAttention 快 56 倍，意味着它确实实现了**亚二次方的计算复杂度**，而不仅仅是常数因子的优化。

## 三、独立评测：Appen 报告深度解读

### 3.1 Appen 是谁？

Appen 是全球知名的 AI 数据服务和模型评估公司，长期为各大模型厂商提供第三方评测服务。Appen 不是学术机构，其评估方法论的商业性质意味着它更关注实用性能而非学术严谨性。但作为独立的第三方，其结果至少提供了**非自说自话的验证**。

### 3.2 关键评测结果

| 测试 | SubQ 成绩 | 对比参照 | 解读 |
|------|-----------|----------|------|
| **LiveCodeBench** | 89.7% | 与顶尖编码模型处于同一水平 | 编码能力达到前沿，这是 SubQ 最强的领域 |
| **速度测试** | FlashAttention 的 56 倍 | 理论基线速度 | 亚二次方复杂度的直接证据 |
| **RULER 128** | 成功完成 | Anthropic Claude: $2600 vs SubQ: $8 | 325 倍的成本差异，但需确认模型规模和输出质量是否可比 |
| **Needle-in-a-Haystack** | 6M/12M tokens 下 98% | 极少模型在如此长的上下文上测试过 | 长上下文检索能力突出 |

### 3.3 数据中的疑点

独立评测虽然提供了验证，但也暴露了一些值得关注的问题：

**1. LiveCodeBench 89.7% 的绝对含义**

89.7% 听起来很高，但需要放在上下文中理解。LiveCodeBench 是竞争编程基准，不同模型的分数差异可能在 1-2% 以内。**89.7% 是"前沿水平"，但不一定是"最顶尖"。** 需要对比同期其他模型的分数才能准确定位。

**2. RULER 128 的成本对比公平性**

$2600 vs $8 的差异令人震惊，但这个比较需要更多细节：
- 两个模型的参数量是否相同？
- 输出质量（答案准确度）是否可比？
- $8 是一次推理成本还是包括模型加载？
- Claude 在 RULER 128 上的运行配置是什么？

如果 SubQ 是小模型而对比的是大模型，成本差异就不完全来自注意力机制的改进。

**3. 400 文档推理 demo**

Whedon 在 MIT Technology Review 的采访中演示了 SubQ 在 400 份文档上执行推理任务，"几秒钟就给出了响应"。作为对比，Perplexity 无法加载全部 400 份文档。这个演示直观地展示了长上下文能力的差异，但：
- 文档的大小和复杂度未知
- "推理任务"的具体内容未公开
- Perplexity 的限制可能来自其系统设计（而非底层模型）

## 四、深度分析：如果 SubQ 是真的，意味着什么？

### 4.1 对 LLM 架构范式的颠覆

如果 Subquadratic 真的实现了在质量不降的前提下将注意力复杂度从 O(n²) 降低到亚二次方，这将是自 2017 年 Transformer 论文以来**最重要的架构创新**。

```
┌──────────────────────────────────────────────────────────────┐
│  注意力机制演进路线图                                          │
├──────────────────────────────────────────────────────────────┤
│                                                               │
│  2017  Transformer (Vaswani et al.)                           │
│    │  O(n²) 密集注意力                                        │
│    ↓                                                          │
│  2019-2023  各种稀疏/线性/核近似方法                            │
│    │  降复杂度但牺牲质量 ← 这是"不可能三角"                      │
│    ↓                                                          │
│  2022-2023  FlashAttention                                    │
│    │  不改变复杂度，只优化常数因子                               │
│    ↓                                                          │
│  2026  SubQ（如果成功）                                        │
│       O(n^α) where α < 2，质量不降                              │
│       ← 打破不可能三角                                          │
│                                                               │
└──────────────────────────────────────────────────────────────┘
```

### 4.2 对行业格局的影响

如果 SubQ 的技术被证实有效，以下连锁反应几乎是必然的：

**1. 前沿实验室的快速跟进**

Google、Meta、OpenAI、Anthropic 拥有最顶级的研究团队和计算资源。如果动态稀疏注意力的基本思路是正确的，这些团队很可能在几个月内复现并改进它。**Subquadratic 的真正优势可能不在于技术本身，而在于先发优势和专利壁垒。**

**2. 开源社区的爆发**

Subquadratic 尚未开源 SubQ 模型。一旦其技术细节被泄露或独立复现，开源社区（Hugging Face、Together AI、Mistral 等）会迅速跟进。参考 FlashAttention 的路径：从 Stanford 的学术论文到成为所有主流推理框架的标准组件，只用了不到一年。

**3. 边缘 AI 的新可能性**

当前，在边缘设备（手机、笔记本、IoT 设备）上运行 LLM 的主要瓶颈是内存和算力。如果 SubQ 能在保持质量的同时将计算量降低数十倍，**本地运行大模型将从"勉强可行"变为"轻松流畅"**。这可能加速 AI 从云端向边缘的迁移。

**4. 对现有投资的冲击**

数十亿美元投入到 GPU 集群建设中，主要是为了支撑密集注意力的计算需求。如果 SubQ 的方法被广泛采用，**同样的硬件可以支持 10-50 倍的吞吐量**。这将对 NVIDIA 等硬件厂商的长期需求预测产生影响——尽管短期内，训练 SubQ 仍然需要大量算力。

### 4.3 为什么大厂没有率先做到？

这是最大的疑问。如果动态稀疏注意力的思路如此直观——"不是所有 token 对都需要互相看"——为什么 Google、Meta、OpenAI 这些拥有最聪明研究者的公司没有先做到？

几种可能的解释：

**解释 1：大厂的路径依赖。** 前沿模型已经投入了数十亿美元在密集注意力的训练管线和基础设施上。切换到一个全新的注意力机制意味着从头训练，风险极高。Subquadratic 作为初创公司，没有这种历史包袱。

**解释 2：技术突破比直觉更难。** "动态选择重要 token 对"说起来简单，做起来极难。历史上几乎所有尝试都以质量下降告终。Subquadratic 可能找到了一个精巧的算法设计，而这个设计并非显而易见。

**解释 3：大厂其实在做，但还没到可以公布的阶段。** 这完全是可能的。大型研究项目通常需要更长的周期来验证和打磨。

**解释 4：SubQ 的成功部分来自工程优化而非纯算法创新。** 动态稀疏选择、模型压缩、训练策略优化等因素可能共同作用，产生了远超单一算法改进的效果。

## 五、 scepticism 有理有据：需要回答的关键问题

### 5.1 权重复用与"全新架构"的矛盾

Subquadratic 明确表示 SubQ 复用了 Qwen 的权重来引导训练。这是一个**严重削弱其"完全重新发明 LLM"宣称的事实**。

如果 SubQ 的底层注意力机制与 Qwen 完全不同（密集 vs 稀疏），那么直接复用权重的意义是什么？可能的解释是：
- SubQ 使用了某种**渐进式迁移训练**，先让模型在密集注意力的权重上预训练，然后逐步切换到稀疏注意力并微调。
- 或者 SubQ 的"稀疏"并不是完全不同的机制，而是对现有注意力机制的某种**参数化改造**。

无论如何，这意味着 SubQ 不是从零开始的"新架构"，而是对现有架构的改良——这并不降低其价值，但确实降低了其革命性。

### 5.2 未公开的"secret sauce"

Subquadratic 拒绝公开动态稀疏选择的具体算法。在 2026 年的开源 AI 社区，这种保密姿态是**极为反常的**。几乎所有重要的模型创新（FlashAttention、GQA、RoPE、SwiGLU 等）都通过论文或开源代码公开了细节。

保密的可能原因：
- **专利保护：** 公司可能正在申请专利
- **竞争优势：** 害怕被大厂快速复现
- **算法尚不成熟：** 方法可能还在迭代中，过早公开会暴露缺陷
- **算法效果被夸大：** 最坏的情况

**在算法细节公开之前，任何关于 SubQ 技术真实性的判断都只能是推测。**

### 5.3 "AI Theranos"的可能性

Dan McAteer 在 X 上的评价——"要么是 Transformer 以来最大的突破，要么是 AI 版 Theranos"——虽然耸人听闻，但并非没有道理。

Theranos 的核心问题是：**在技术细节被独立验证之前，宣称的能力无法被证伪。** Subquadratic 目前的处境与之有相似之处：
- 有大量令人印象深刻的数字（56×、$8 vs $2600、12M tokens）
- 但关键的技术细节未公开
- 第三方评测有限且不透明
- 模型尚未广泛可用

当然，Subquadratic 和 Theranos 有一个根本区别：**SubQ 至少已经有了第三方评测（Appen），而且这些评测结果看起来是合理的。** Theranos 的血液检测设备从未通过任何独立验证。

## 六、更广阔的图景：注意力机制的未来

### 6.1 三条并行的优化路径

无论 SubQ 最终被证实还是被证伪，注意力机制的优化正在从三个方向同时推进：

```
┌──────────────────────────────────────────────────────────────┐
│  注意力优化的三条路径                                          │
├──────────────────────────────────────────────────────────────┤
│                                                               │
│  路径 1：工程优化（已有成熟成果）                               │
│    • FlashAttention V2/V3：I/O 感知优化                        │
│    • PagedAttention：分页内存管理                               │
│    • KV Cache 量化：INT8/INT4 存储                              │
│    → 降低常数因子，不改变复杂度                                 │
│                                                               │
│  路径 2：架构优化（SubQ 的方向）                                │
│    • 动态稀疏注意力：Subquadratic                               │
│    • 线性注意力：Performer 等                                   │
│    • 状态空间模型：Mamba/RWKV                                   │
│    → 改变复杂度，但质量 trade-off 是核心挑战                     │
│                                                               │
│  路径 3：混合架构（可能的最终方向）                             │
│    • 局部密集 + 全局稀疏：类似 Longformer 但更智能              │
│    • 注意力 + SSM 混合：Jamba（AI21 Labs）                      │
│    • 多层不同策略：近层密集，远层稀疏                            │
│    → 兼顾质量和效率的折中方案                                   │
│                                                               │
└──────────────────────────────────────────────────────────────┘
```

### 6.2 一个被忽视的方向：注意力之外的架构

在讨论稀疏注意力的同时，值得提一下 **Mamba 和 RWKV 这类完全摒弃注意力机制的架构**。它们使用状态空间模型（SSM）或循环机制来实现线性复杂度的序列建模。

- **Mamba（2023）：** 基于选择性状态空间，线性复杂度，在某些任务上接近 Transformer
- **RWKV（2023-2024）：** 将 Transformer 的并行训练和 RNN 的线性推理结合
- **Jamba（2024）：** AI21 Labs 的混合架构，交替使用注意力层和 Mamba 层

这些方法代表了比稀疏注意力更激进的范式转换。如果 SubQ 的动态稀疏注意力最终被证明可行，那说明注意力机制本身还有很大的优化空间。如果不行，那么 Mamba/RWKV 这类替代架构可能会更快地成为主流。

### 6.3 2026 年的信号：效率时代正在到来

MIT Technology Review 的报道中引用了 Subquadratic CEO Justin Dangel 的话：

> *"我们认为几年内不会再有人基于 Transformer 构建模型。"*

这番话无论是否准确，都指向一个明确的行业信号：**LLM 的竞争正在从"谁的能力更强"转向"谁的效率更高"。**

这个转向有几个驱动因素：

1. **成本压力：** 训练和运行前沿模型的成本已经让大多数公司望而却步。NVIDIA 的 RULER 128 测试花费 $2600 就是一个缩影。
2. **能耗约束：** 数据中心的能源消耗已经成为环境和社会问题。更高效的模型意味着更少的电力需求。
3. **边缘部署需求：** 越来越多的应用场景需要本地运行 AI，而不是依赖云端 API。
4. **长上下文需求：** Agent 工作流需要处理整个代码库、数百份文档、超长对话历史，这直接推动了对更大上下文窗口的需求。

## 七、结论：谨慎乐观，等待开源

Subquadratic 的 SubQ 模型提出了一个极具吸引力的愿景：打破 Transformer 的二次方瓶颈，实现高效、低成本、超长上下文的 LLM。Appen 的第三方评测提供了初步的支持证据，但关键的技术细节仍然保密。

**我的判断：**

1. **SubQ 的技术很可能是真实的**——56 倍速度提升和长上下文能力在独立评测中得到了验证，这些不太可能是完全捏造的。
2. **但"终结 Transformer"的宣称过于激进**——权重复用 Qwen 意味着 SubQ 至少部分依赖于现有的 Transformer 训练成果，而非完全从零构建。
3. **真正的考验是开源和复现**——在 Subquadratic 开源 SubQ 的模型权重和训练代码之前，学术界和工业界无法全面评估其技术贡献。
4. **即使 SubQ 不是"革命"，它也是一次重要的"演进"**——动态稀疏注意力的思路如果被验证有效，将为注意力机制的优化开辟一条新路径，无论最终由谁来实现。

**对于开发者的建议：** 关注 Subquadratic 的后续进展，但不要在 SubQ 开源之前就对其过度依赖。同时，继续投资 FlashAttention、KV Cache 优化等已验证的技术——这些在短期内仍然有最高的投入产出比。

**对于研究者的建议：** 尝试独立复现动态稀疏注意力的思路。即使无法完全达到 SubQ 的效果，对这个问题的探索本身就极有价值。历史上，即使"失败"的注意力优化尝试（如 Performer）也为后续研究提供了重要的理论基础。
