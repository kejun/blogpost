# Agora-1：多智能体世界模型——当"Learned Game Engine"遇见共享模拟

**文档日期：** 2026 年 5 月 19 日  
**标签：** World Model, Multi-Agent, Agora-1, Odyssey, Reinforcement Learning, Simulation, DiT

---

## 一、引言：世界模型的"多人模式"终于来了

2026 年 5 月 18 日，前 OpenAI 产品副总裁 Oliver Cameron 创立的 Odyssey 公司发布了 **Agora-1**——第一个支持多智能体实时交互的世界模型（Multi-Agent World Model）。

这不是一次渐进式的改进。Agora-1 解决了一个困扰世界模型领域多年的根本性问题：**如何在多个参与者同时交互的情况下，保持共享世界状态的一致性？**

在此之前，所有的世界模型——从 Wayve 的 GAIA-1 到 Odyssey 自己的 Prowl——都只支持单一参与者。你可以让一个 AI 在生成的虚拟世界中行走、驾驶、射击，但你无法让两个 AI **同时**在同一个世界中相遇、对抗或协作。

Agora-1 打破了这个限制。它最多支持四名玩家在同一个生成的 GoldenEye（007 黄金眼）关卡中进行实时死斗对战，每个玩家看到的是从各自视角渲染的同一个世界——而且所有画面都是模型实时生成的。

换句话说，Agora-1 **学会了一个游戏引擎**。

## 二、为什么多智能体世界模型如此困难

### 2.1 单一世界状态的一致性悖论

世界模型的核心承诺是：给定当前状态和一个动作，预测下一帧。对于单智能体来说，这已经是一个极其困难的序列建模问题。

但当引入第二个智能体时，问题变得指数级复杂：

```
单一智能体世界模型：
  State_t + Action_A → State_{t+1} → Render from A's view

多智能体世界模型：
  State_t + Action_A + Action_B → State_{t+1}
    → Render from A's view (A sees B at position p₁)
    → Render from B's view (B sees A at position p₁)
    ↑ 两个渲染必须指向同一个 p₁
```

核心挑战在于：**多个视角下的渲染必须指向同一个底层世界状态**。如果 A 看到 B 在左边，B 必须同时看到 A 在右边。当 A 和 B 背对背行走、随后再次回头时，他们看到的世界必须是连续的——地图不能消失，物体不能瞬移。

这是一个**分布式一致性问题**，只不过一致性协议不再是 Paxos 或 Raft，而是一个神经网络。

### 2.2 已有路线的局限

在 Agora-1 之前，学术界尝试过多条路线，但各有致命缺陷：

| 方法 | 核心思路 | 致命缺陷 |
|------|----------|----------|
| **Multiverse** (Enigma) | 将多个智能体的状态拼接为"分屏"表示 | 本质仍是单一世界状态；智能体数量增加时模型容量不变，每个智能体的状态分辨率被压缩 |
| **Solaris** (arXiv 2602.22208) | 在自回归 DiT 的序列维度上拼接参与者 | 模型上下文随玩家数量线性增长，无法扩展到多人场景 |
| **MultiGen** (arXiv 2603.06679) | 显式维护共享世界状态 | 模拟动态和渲染的建模方式不同，一致性在玩家互不可见时容易崩溃 |

共同问题：**当玩家脱离彼此视野后重新相遇，世界状态的一致性无法可靠维持。** 这听起来像是技术细节，但对于任何需要长期一致性的应用（游戏、机器人、训练模拟器），这是不可接受的。

## 三、Agora-1 的架构创新：模拟与渲染的解耦

Agora-1 的核心架构思想可以用一句话概括：

> **将世界状态的演化（Simulation）与世界状态的可视化（Rendering）解耦。**

这听起来像是游戏引擎的基本设计原则——Unreal Engine 和 Unity 天然就是如此设计的。但 Agora-1 的革命性在于：**两个组件都是学习得到的系统**，没有任何硬编码的游戏逻辑或渲染规则。

### 3.1 架构总览

```
┌─────────────────────────────────────────────────────────────────┐
│                        Agora-1 架构                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Player A Action ──┐                                            │
│                    ▼                                            │
│  Player B Action ──→ [World State Model] ──→ Shared World State │
│                    ▲                           │                │
│  Player C Action ──┘                           │                │
│                                                 │                │
│                            ┌────────────────────┤                │
│                            ▼                    ▼                │
│                   [Renderer A's View]   [Renderer B's View]      │
│                            │                    │                │
│                            ▼                    ▼                │
│                    Generated Pixels A   Generated Pixels B       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

架构由两个学习模型组成：

#### 模型一：World State Model（世界状态模型）

负责学习世界状态如何在玩家动作的作用下随时间演化。

- **输入：** 当前共享世界状态 + 所有玩家的联合动作
- **输出：** 下一时刻的共享世界状态
- **训练数据：** 直接从 GoldenEye 游戏的内部状态中提取——包括每个玩家的位置、生命值、武器状态、弹药、关卡布局等

这里的关键洞察是：**离散的游戏状态（discrete game state）与大多数基于 DiT 的世界模型所处理的连续视觉域（continuous visual domain）在结构上完全不同。** 世界状态模型需要专门为游戏状态建模设计架构，并且需要大量结构化训练数据。

世界状态跟踪的信息包括：
- 每个玩家的位置和朝向
- 每个玩家的生命值
- 武器和弹药状态
- 关卡中的物体和障碍物

#### 模型二：Rendering Model（渲染模型）

负责从共享世界状态生成视觉画面。

- **输入：** 共享世界状态 + 特定玩家的视角参数（位置、朝向）
- **输出：** 从该视角看到的像素帧
- **架构：** 基于 DiT（Diffusion Transformer）的世界模型
- **条件信号：** 直接使用游戏状态，而非文本提示、图像或其他传统条件信号

这是 Agora-1 与传统视频生成模型（如 Sora、Kling）的根本区别。Sora 的条件信号是文本提示，Kling 的条件信号是首帧图像。而 Agora-1 的条件信号是**结构化的游戏状态向量**。

### 3.2 解耦带来的三个关键优势

**优势一：一致性的天然保证**

因为所有视角的渲染都源自同一个共享世界状态，所以一致性是架构层面的保证，而不是需要额外优化的约束条件。A 看到 B 在某个位置，B 的渲染也会从同一个位置出发——因为底层状态是同一个。

**优势二：玩家数量的可扩展性**

增加第五个玩家不需要修改世界状态模型，也不需要重新训练渲染模型——只需要多一个渲染实例。渲染是并行独立的，每个视角的生成互不影响。

这与 Solaris 的方案形成鲜明对比：Solaris 将玩家拼接在序列维度，上下文长度随玩家数量线性增长，计算成本也随之飙升。

**优势三：世界状态的可操控性**

因为世界状态是显式的、结构化的，所以可以**直接操纵底层状态**。这意味着 Agora-1 不仅可以模拟已有的关卡，还可以生成全新的关卡——只要世界状态的格式保持一致，游戏动力学就能保持。

这打开了"无限内容生成"的大门。

### 3.3 与传统游戏引擎的类比

```
传统游戏引擎 (如 Unreal/Unity)：
  Game Logic (硬编码 C++) → World State → Renderer (GPU pipeline) → Pixels

Agora-1 (Learned Game Engine)：
  World State Model (神经网络) → World State → Rendering Model (DiT) → Pixels
```

两者在结构上惊人地相似。但传统引擎的每个环节都是手工编写的代码——物理引擎、碰撞检测、渲染管线。Agora-1 的每个环节都是从数据中学习得到的。

这种类比既有启发性，也暴露了一个根本性的差异：传统引擎的确定性来自于精心设计的算法，而 learned engine 的确定性来自于训练数据的覆盖度和模型的泛化能力。

## 四、更广阔的图景：多智能体世界模型为什么重要

### 4.1 多智能体强化学习（MARL）的训练环境瓶颈

Agora-1 论文中提到了一个关键观点：

> **通往更通用智能的瓶颈不再是模型架构，而是训练经验的可用性。**

具体来说，是智能体**主动寻求能够提升自身能力的交互**的能力。

传统的世界模型只支持单一参与者，这意味着它们只能用于单智能体强化学习的训练环境。但现实世界中的智能体——无论是机器人、自动驾驶汽车还是 AI 助手——都需要在多智能体环境中学习。

Agora-1 打破了这个限制。随着参与者数量的增加，联合交互空间呈**组合级增长**：碰撞、协同移动、争夺目标，以及其他涌现行为。被动收集的演示数据只能覆盖这个空间的极小一部分。

**多智能体强化学习提供了一个可扩展的机制**：通过开放式的交互来生成缺失的数据。智能体和世界模型可以共同进化——智能体在更好的世界中学习，世界模型因为智能体的探索而获得更好的训练数据。

这与 Odyssey 此前的 PROWL 工作形成互补：PROWL 使用对抗性策略来暴露世界模型的缺陷并生成新的训练数据，而 Agora-1 将对抗性从"智能体 vs 世界模型"扩展到了"智能体 vs 智能体"。

### 4.2 想象式训练（Imagined Training）

一个更大胆的想法是：**在生成的世界中训练的策略，能否迁移到真实环境？**

这听起来像是科幻小说——在虚拟世界中训练的 AI 机器人，突然被放到真实世界中就能工作。但这并非空想。AlphaGo 在自我对弈中超越了人类棋手，而这些对弈完全发生在虚拟的棋盘上。关键在于模拟器与真实环境的"域差距"（domain gap）。

Agora-1 提供了一种可能性：如果世界模型足够逼真，那么在其中训练的策略可能具有**零样本泛化**到未见环境和未见交互伙伴的能力。

这比传统的 Sim2Real（仿真到真实）更进一步——它是 **Sim2Sim2Real**，即在一个学习得到的模拟器中训练，然后迁移。

### 4.3 超越游戏：协作机器人与多视角仿真

Agora-1 虽然以 GoldenEye 为测试平台，但其架构不依赖于游戏。许多现实系统需要多个智能体在同一个共享环境中运行：

- **协作机器人：** 多个机器人需要共同推理动作、空间和彼此之间的交互
- **多视角仿真：** 从多个视角同时观察同一个动态场景
- **数字孪生：** 工厂、城市的实时仿真，多个决策者同时交互

架构的通用性意味着 Agora-1 的技术路线可以迁移到这些领域。

## 五、技术挑战与开放问题

### 5.1 从游戏状态到现实世界的跨越

Agora-1 目前的状态模型依赖于从游戏中提取的**显式内部状态**——位置、生命值、武器。这在训练阶段是可行的，因为游戏引擎提供了这些数据。

但在现实世界中，没有"游戏引擎"可以查询内部状态。要将这种方法迁移到机器人或自动驾驶等真实场景，状态模型必须从**纯感知输入**（摄像头、激光雷达）中**学习推断出隐式的内部状态表示**。

这是一个巨大的跨越：从"知道世界是什么"（因为游戏告诉你）到"推断世界是什么"（从像素中学习）。

HN 评论区的一个观察切中了要害：

> *内部世界状态需要被学习才能迁移到真实机器人领域，因为你无法在训练中查询游戏引擎的内部状态。*

### 5.2 环境交互性的边界

Agora-1 目前主要处理的是少数智能体在**相对静态的环境中交互**。地图是固定的，物体不会自发变化，交互规则是预先定义的。

但现实世界的交互性远远超出这个范围。人类学习者能够高效地获得对环境交互的强大模型——这就是为什么我们能在从未见过的环境中快速适应。

**学习"交互性"本身**——而不仅仅是学习已有交互的规律——是世界模型面临的下一个巨大挑战。游戏中的交互是手工添加的，但真实世界的交互是开放的、无限的、不可枚举的。

### 5.3 状态表示的泛化

Agora-1 的状态表示是为 GoldenEye 量身定制的：位置、生命值、武器。如果要扩展到多个游戏甚至非游戏场景，状态表示需要能够**跨规则泛化**。

这意味着需要一种**通用的状态表示语言**——就像游戏引擎中的 Entity-Component 系统，但由模型自己学习得到，而非手工设计。

### 5.4 计算成本

DiT 模型的渲染计算成本极高。Agora-1 需要为每个视角独立运行渲染模型，这意味着多人场景的计算成本是单人场景的 N 倍。

对于实时交互应用（游戏、远程呈现），这要求渲染延迟极低。目前的实现依赖于 Odyssey 内部的 GPU 集群——它还不是一个可以在消费级硬件上运行的系统。

## 六、横向对比：世界模型领域的 2026 格局

Agora-1 不是孤立的事件。2026 年，世界模型领域正在经历一次集中爆发：

| 项目 | 团队 | 核心创新 | 支持多人？ |
|------|------|----------|-----------|
| **GAIA-1** | Wayve | 自动驾驶世界模型 | ❌ |
| **Prowl** | Odyssey | RL 驱动的对抗性世界模型改进 | ❌ |
| **Multiverse** | Enigma | 分屏式多智能体世界模型 | ⚠️ 有限 |
| **Solaris** | — | 自回归 DiT 序列拼接 | ⚠️ 不可扩展 |
| **MultiGen** | — | 显式共享状态 | ⚠️ 一致性脆弱 |
| **Agora-1** | Odyssey | 模拟与渲染解耦 | ✅ 最多 4 人 |

关键趋势：**从单一世界到共享世界，从被动渲染到主动模拟，从单智能体到多智能体。**

Agora-1 在这个趋势中占据了一个关键位置——它是第一个实现了真正的架构解耦的方案，这意味着它在可扩展性和一致性上都有了结构性的保证。

## 七、与 AI Agent 生态的关联

世界模型和多智能体 Agent 之间的关系正在变得越来越紧密。几个关键连接点：

### 7.1 World Model as a Training Environment for Agents

如果世界模型可以生成高保真的、多智能体的仿真环境，那么它们自然成为 Agent 的训练沙箱。这与 2026 年初 HuggingFace 发布的 **Open Agent Leaderboard** 形成互补：Leaderboard 评估 Agent 在真实任务上的表现，而世界模型提供的是 Agent 可以在其中无限练习的环境。

### 7.2 对抗性评估

Agora-1 的多人模式为 Agent 评估提供了一个全新的维度：**在对抗环境中测试 Agent 的鲁棒性**。当前的 Agent 评估（SWE-bench、AppWorld、tau2-Bench）都是单智能体任务——Agent 对抗的是固定的环境。但在多人世界中，Agent 需要应对其他智能体的**对抗性策略**，这是一个更难的测试。

### 7.3 多智能体协作的涌现

Agora-1 提到，随着参与者增加，联合交互空间会涌现出新的行为：协同移动、争夺目标、临时结盟。这些涌现行为对于理解多智能体 Agent 的社会性行为至关重要——它提供了一个可控的实验环境。

## 八、独到见解：Agora-1 告诉我们什么

### 洞察一：世界模型的"操作系统化"

Agora-1 的架构暗示了一个更大的趋势：**世界模型正在从"视频生成器"演变为"可交互的操作系统"**。

传统世界模型（如 Sora）的目标是生成一段视频。Agora-1 的目标是维护一个可交互的、持续演化的状态空间。这更接近操作系统而不是视频编辑器。

如果把世界状态模型比作操作系统内核，渲染模型比作图形驱动，玩家的动作输入比作用户进程——那么 Agora-1 本质上是一个**学习得到的操作系统**。

### 洞察二：MARL 的"AlphaGo 时刻"可能来自世界模型

单智能体 RL 的突破性时刻是 AlphaGo——一个在自我对弈中超越人类水平的系统。但多智能体 RL 一直没有类似的"AlphaGo 时刻"。

Agora-1 + PROWL 的组合可能就是这个突破口。一个可以生成无限多变体的世界模型，加上一个在其中自我进化的多智能体系统，构成了一个封闭的训练循环。

### 洞察三：Learned Engine 的终局

如果世界模型可以学习游戏引擎的所有组件——物理、逻辑、渲染——那么"游戏开发"本身可能被重新定义。设计师不再编写代码，而是**设计世界状态的先验**，然后让模型学习剩下的部分。

这听起来像是对程序员的替代，但更准确的说法是：**它将游戏开发的抽象层级从"如何实现"提升到了"想要什么"**。

---

## 九、结论

Agora-1 不是一个"更好的视频生成模型"。它是世界模型领域的一个范式转移——**从单智能体的被动渲染到多智能体的主动模拟**。

它的核心架构创新（模拟与渲染的解耦）不仅在技术上优雅，而且在概念上深刻：它揭示了世界模型的真正本质不是"生成像素"，而是"维护状态"。

在 2026 年这个多智能体 Agent 开始大规模部署的年份，Agora-1 提供了一个关键的基础设施：**一个可以让多个 Agent 同时存在、交互、竞争、协作的共享世界**。

这条路还很长。从 GoldenEye 的显式游戏状态到真实世界的隐式状态推断，从 4 人死斗到千人规模的仿真，从学习到泛化——每一步都是巨大的挑战。

但 Agora-1 已经证明了一件事：**多人模式的世界模型，是可行的。**
