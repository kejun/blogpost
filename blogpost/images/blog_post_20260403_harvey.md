## 一个周末的实验

上周末，我让我的父母体验了 Agent 技术（agent-pilled）。他们都已退休，空闲时间里我爸爸玩纸牌游戏，我妈妈为一个开源科学计算库做贡献。几个月来我一直告诉他们应该试试新的编码模型，比如 [Claude Code](https://docs.anthropic.com/en/docs/claude-code/overview) 或 [Codex](https://openai.com/codex)。这个周末我终于坐下来教他们怎么用。

我妈妈问了一个简单的问题：这些系统能不能帮她提高所贡献库的测试覆盖率？

我告诉她，系统可以获取她的代码库，与类似的科学计算库进行比较，找出覆盖率差距，推断更成熟的项目会有哪些测试，编写计划，实现这些测试，运行项目，调试失败，并持续迭代直到成功。她笑了，因为这听起来太荒谬了。

**然后我们试了试。五分钟后，系统返回了一份详细的同类库调研和具体实施计划。十五分钟后，它已经编写了测试、构建了项目、运行了测试套件、发现了现有测试中的 bug，并持续迭代，直到 C++、MATLAB 和 Julia 的所有测试都通过。**

然后我爸爸问它能不能实现 deferred corrections 和 VARPRO——这是他在研究生涯中帮助开创的算法——并添加同样的测试覆盖。我们也试了，**它成功了**。模型找到了这些算法，实现了它们，测试了它们，并将它们纳入库的约定中。

我妈妈说第一个任务她至少要花一个月时间。我父母都惊呆了。

## 能力飞跃被低估了

尽管我父母的背景，他们完全被**过去几个月的能力飞跃**打了个措手不及。他们都有计算机科学博士学位。我妈妈在 Apple 工作了 30 年，领导了一个大型团队开发 autocorrect——这是语言模型最早的十亿用户应用之一。我爸爸是斯坦福教授，研究了许多后来成为神经网络基础的科学计算方法。他们住在硅谷。他们有两个儿子在 Harvey 工作。我一直在不停地谈论这项技术。他们每天都用 ChatGPT。**但他们仍然被编码 Agent 的水平完全震惊了。**

如果你不是全职使用这些模型的工程师，很难理解即将到来的变化有多大。

为他们辩护一下，我父母都退休了。但更重要的点是：**世界上大多数人即将有和他们一样的反应。**

当我们在早期律所和企业客户面前展示现在 Agent 能做什么时，我一直在看到这种反应：[可以像一组初级律师一样在整个客户案件上运作的系统](https://www.harvey.ai/blog/the-year-legal-ai-becomes-core-infrastructure)，或者以有意义的自主性处理合同谈判。反应通常是某种版本的难以置信。

**上一次感知差距这么大还是从 GPT-3 到 GPT-4。** 那时，惊讶在于模型已经变得足够好，可以改变一个人能做什么。**这一次，后果是组织本身开始改变。**

## 从个人杠杆到组织杠杆

在过去的几年里，基本模式很清晰：模型坐在工程师旁边，让那个工程师更快。**人类保持在循环中心**，在每一步决定下一步做什么并引导系统。

**现在循环正在改变。** 你可以给 Agent 一个目标、正确的上下文、正确的工具和正确的约束，它可以检查代码库、制定计划、编写代码、运行测试、调试失败、从错误中恢复，并独立工作数小时。

**杠杆不再局限于一个人工作得更快。它开始上升到一个新的层次，从个人到组织。**

## 组织作为信息路由层次结构

[大型组织一直是作为信息路由层次结构构建的](https://sequoiacap.com/article/from-hierarchy-to-intelligence/)。管理者聚合上下文、路由决策、跟踪阻塞点、保持团队对齐，因为信息历来必须通过人流动。

**自主 Agent 开始直接承担部分协调功能。** 它们不仅执行任务。它们监控系统、跨团队携带上下文、触发工作、浮现决策。

**这就是为什么变化比生产力提升更大。它改变了组织运行的协调层。**

## 为什么工程是第一个

工程是第一个变得不可否认的地方，因为**软件已经生活在机器可读的循环中**。指令是数字化的，工具是数字化的，环境是数字化的，输出可以被其他机器测试。

实验室也有充分理由首先让模型擅长代码，因为**代码是构建下一代这些系统的方式**。

这就是为什么工程正在成为第一个围绕 Agent 重组的职能，为什么你已经在系统中看到这种模式，比如 Ramp 的 [后台 Agent](https://engineering.ramp.com/post/why-we-built-our-background-agent) 和 Stripe 的 [端到端编码 Agent](https://stripe.dev/blog/minions-stripes-one-shot-end-to-end-coding-agents)。

**工程是未来杠杆首先出现的地方，因为工作已经以 Agent 可以直接进入的方式结构化。**

## Harvey 的 Spectre 系统

我们目前在 Harvey 正在经历这种转型。我们构建了一个名为 **Spectre**（以 Dota 2 角色命名）的内部 Agent 系统，它开始自主处理越来越多的工程工作，以及越来越多的非工程工作。

**它做的大部分工作不再由人类提示触发。它由系统监控公司并基于事件、bug 报告、客户反馈和 Slack 消息做出决策来触发。**

> "在实践中，Spectre 是公司世界模型的开端：一张 Harvey 内部正在发生什么以及接下来需要发生什么的实时图景。"

我们的工程师现在效率如此之高，以至于更难协调。**瓶颈正从实施转向审查、优先级排序、协调和运营设计。**

**这就是组织内部新杠杆的样子：可以发生的工作比旧协调结构能吸收的更多。**

## 无限 AI 员工的时代

**随着能够雇佣无限 AI 员工，公司将不再受吞吐量限制。** 随着员工单独行动的速度趋于渐近线，机构需要重新学习如何一起走远。

这需要从根本上重新思考：
- 什么工作重要
- 如何审查工作
- 如何信任工作
- 如何围绕它培训人员
- 如何定价
- **如何围绕被判断瓶颈的智力盈余重新设计组织**

> "在这种条件下，有意义的杠杆不再是一个组织能生产多少。杠杆在于人、团队和机构能在人类和 Agent 之间协调多少上下文。"

即使对于 AI 原生公司，这也很难。

## 法律行业的结构性变革

随着 Agent 能力快速发展，为当前能力做计划可能感觉无用且过时。然而，从我们的视角，我们可以看到清晰且持久的转变，AI 将如何影响法律——既作为 Agent 的消费者，也作为 Agent 如何在组织中实施的关键利益相关者。

与其他行业一样，**法律 Agent 将开始挑战律所的结构惯例**。

律所是深度层次化的，使用合伙人和初级律师之间的报告链来将有限的法律专业知识资源分配到极其复杂的案件中。层次结构中较初级的部分专注于吞吐量——组织大量数据或执行基本常规任务。

**随着这些任务越来越多地委托给 Agent，智力取代层次结构。**

> "智力取代层次结构"

**每个律师现在因其判断而非产出而珍贵；要求律所重新思考人员配置、学徒制、定价、实践领域结构以及与客户合作的方式。**

---

## 案件级别的世界模型

我们预计这些趋势将在案件级别出现。

**每个案件及其相关文档、消息、研究、工作流和其他数据可以类比为独立的世界模型，AI Agent 团队可以在其中运作以改变法律实践。**

这种转型不会取代律师，但它确实改变了：
- 案件如何协调
- 判断如何应用
- **杠杆在哪里可以找到——对律所和企业团队都是如此**

**更多吞吐量从根本上意味着更多判断调用，以及对不仅高技能而且高信任律师的更深需求。**

## 企业法务团队的新角色

对于企业团队，Agent 的扩散要求他们不仅要驾驭直接工作中的转型，还要**作为整个组织有效 AI 实施的管家**。

自然地，人机组织生产力的提高导致：
- 政策问题增加
- IP 和产品审查增加
- （潜在）事件增加

**法律团队需要找到杠杆来有效处理这种数量。**

> "对于企业团队，Agent 的扩散要求他们不仅要驾驭直接工作中的转型，还要作为整个组织有效 AI 实施的管家。"

## 法律治理 Agent 部署

此外，**法律将越来越多地被要求治理公司其他部分如何使用 Agent。**

虽然工程将定义 Agent 的能力，**法律将治理这些能力如何安全部署**：
- 责任在哪里
- 风险如何管理
- 什么风险是可容忍的
- 如何在公司内赢得信任

**通过划定组织可以依赖 Agent 的界限，企业团队将从根本上定义新杠杆方程的边界。**

> "随着吞吐量不再是有意义的约束，核心问题不再是人们应该做什么，而是我们如何围绕智能组织并治理结果。"

## 法律定义可信采用

**法律将是受 Agent 转型影响最大的行业之一，但也将是决定这项技术是否对社会有利的最重要行业之一。**

随着吞吐量不再是有意义的约束，核心问题不再是人们应该做什么，而是我们如何围绕智能组织并治理结果。**这些问题既是法律的，也是技术的。**

作为早期和关键采用者，**律所和企业团队将定义可信采用的样子**：
- 责任在哪里
- 什么风险是可接受的
- 需要什么治理
- 在真实机构内依赖自主系统意味着什么

## 核心洞察总结

### 1. 杠杆的演变

| 时代 | 杠杆来源 | 限制因素 |
|------|---------|---------|
| 工业时代 | 机械放大体力 | 能源、设备 |
| 软件时代 | 代码复制智力 | 工程师数量 |
| AI 时代 | Agent 自主执行 | **判断、协调、信任** |

### 2. 组织转型路径

```
个人效率提升 → 团队重组 → 协调层变革 → 组织重新设计
     ↓              ↓           ↓            ↓
  Copilot      人机协作    Agent 协调    智力盈余管理
```

### 3. 法律行业特殊性

- **高度层次化**：合伙人→初级律师的知识和任务分配
- **判断密集型**：高价值工作依赖经验和直觉
- **信任关键**：客户关系建立在信任和问责上
- **监管密集**：合规、责任、风险管理是核心

### 4. Harvey 的 Spectre 系统启示

- **触发方式**：从人类提示→系统监控和决策
- **工作范围**：从工程→非工程扩展
- **公司世界模型**：实时了解公司内部状态和下一步需求
- **新瓶颈**：实施→审查、优先级、协调、运营设计

### 5. 关键问题转变

**旧问题：**
- 人们应该做什么？
- 如何提高吞吐量？
- 如何自动化任务？

**新问题：**
- 如何围绕智能组织？
- 如何治理结果？
- 责任在哪里？
- 什么风险可接受？
- 如何建立信任？

## 对数据库和 AI 基础设施的启示

作为数据库产品设计和开发者，这篇文章揭示了几个关键机会：

### 1. 上下文图谱（Context Graph）的必要性

Agent 需要：
- 跨团队携带上下文
- 监控系统状态
- 触发工作流
- 浮现决策点

**这正是决策追踪和上下文图谱要解决的问题。**

### 2. 世界模型的数据需求

公司世界模型需要：
- 实时事件流
- 跨系统数据整合
- 决策历史追踪
- 结果反馈循环

**数据库需要支持：**
- 低延迟写入和查询
- 多模态数据（文档、消息、代码、决策）
- 时间旅行查询（重放决策时状态）
- 权限推理（permissioned inference）

### 3. 判断捕获系统

随着吞吐量不再是约束，**判断成为稀缺资源**。数据库可以：
- 捕获人类对 Agent 输出的修改
- 存储决策理由和权衡
- 建立判断知识库
- 支持基于先例的推理

### 4. 信任和问责基础设施

法律治理 Agent 需要：
- 审计追踪（谁/什么做了什么决定）
- 可解释性（为什么做这个决定）
- 责任归属（决定可追溯到人或系统）
- 风险管理（风险阈值和升级路径）

**这些都是数据基础设施问题。**
