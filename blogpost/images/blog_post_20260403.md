*消费者平台构建了互联网时代最强大的商业模式之一*

## 核心观点

消费者平台在过去二十年中构建了最强大的商业模式之一，围绕一个**复合循环 (compounding loop)**：每一次用户互动都成为改进系统的信号。Netflix、Meta、Amazon、TikTok 和 Google 不仅仅记录结果，它们以极高的粒度记录了行为——你点击了什么、忽略了什么、在什么上面悬停、放弃了什么、什么让你回来——并将这些信号输入到学习系统中。这个循环（捕获→学习→改进→再捕获）成为互联网时代最伟大的复合资产之一。

**企业软件从未有过类似的循环。** 不是因为企业决策频率更低，而是因为它们更难被观察。

## B2C 与企业软件的根本差异

消费者系统在受控界面内运行，单个用户在完全由公司拥有的产品内行动。企业决策则根本不同：它们是跨销售、财务、法务、运营、安全和管理的**多人谈判**——每个角色都有不同的激励、不同的权限和不同的约束。销售想要速度，财务想要利润率，法务想要先例控制。这些决策是经过协商的，而不仅仅是点击。

迄今为止，企业缺乏对连接行动与结果的推理过程的工具化。

**B2C 公司已经复合行为信号二十年了。B2B 公司基本上没有。现在，这种情况首次开始改变。**

## 为什么 SaaS 估值倍数被压缩

SaaS 估值倍数被压缩，因为 AI 正在商品化那些证明溢价合理的功能层。当 LLM 可以为几乎任何工作流程生成合格的首稿时，"已知流程上更好的 UI"的价值就会崩溃——那些以功能而非数据为护城河的公司正在被下调估值。他们构建了工作流程，但从未构建复合循环。

**问题是什么能取代功能成为企业价值的持久来源。答案是：企业软件从未有过的复合循环，不是建立在行为追踪上，而是建立在决策追踪上。**

## 决策追踪：缺失的层次

**企业系统被设计为记录最终状态，而非推理过程。** 折扣字段告诉你最终数字，而不是为什么这个数字是合理的。修订后的合同告诉你最终条款，而不是沿途拒绝了哪些备选方案。已解决的工单告诉你事件已关闭，而不是为什么选择了一个升级路径而不是另一个。**决策追踪就存在于事件与结果之间缺失的层次中。**

上下文图谱 (Context Graph) 就是当这一层变得结构化、可查询、并跨系统、跨参与者、跨时间连接起来时的产物。

### 过去的三大障碍

1. **相关信号稀疏、碎片化，嵌入在人类工作流程中，而非作为一等遥测数据被捕获**。企业决策部分发生在会议中，部分在某人脑海中，部分在邮件线程中，部分在私下对话中，部分在互不沟通的系统内。

2. **没有理由存储它。** 决策数据被视为过程废气——短暂的、可丢弃的——因为不存在可以从中学到东西的系统。即使捕获了碎片，它们也很少复合。

3. **缺乏结构化提取能力。** 公司有转录、邮件线程、评论和审批，但没有实际方法从中提取结构化的决策工件，跨系统连接它们，并将它们与结果联系起来。

## 三大转变使决策追踪成为可能

### 1. 企业工作现在生活在可工具化的表面上

工作变得分布式和异步。决策越来越多地在评论线程、文档建议、工单历史、审批流和通话录音中做出。曾经只存在于某人头脑中的推理，现在在工作流程本身中留下了越来越丰富的痕迹。

### 2. 语言模型使非结构化数据可计算

多年来，公司有转录、聊天记录、文档评论和工单历史，但这些 mostly 是可搜索的，而非可学习的。现在 LLM 可以从中提取决策工件。语言模型并没有消除对结构或评估的需求，但它们使得将以前惰性的协作数据转化为系统可以推理的东西成为可能。

### 3. Agent 自动创建决策检查点

**这是最重要的转变。** Agent 在工作流程中提出行动，人类批准、修改或升级。Agent 起草定价提案；销售代表将折扣从 25% 调整到 30%，并添加备注："来自供应商 X 的竞争压力，需要匹配他们的报价。" **这个编辑就是决策追踪。**

模型的提案是结构化的先验——系统认为什么是正确的。人类的修改是判断信号——模型遗漏的实际重要内容。随着 Agent 将自己插入更多工作流程，更多判断通过编辑、批准、例外和覆盖变得明确。工具化不再是可选的，它是工作完成方式的副产品。

## 关键架构问题：你在写入路径还是读取路径？

要捕获决策追踪，你必须在决策做出时在场——而不是之后。这是关键架构问题：**你在写入路径 (write path) 还是读取路径 (read path)？**

当决策作为最终状态落入记录系统时，**为什么**已经消失了。战略表面是决策变得具有约束力的点：审批步骤、修订、升级、Agent 提案、人类覆盖。那是推理仍以可用形式存在的地方。

## 各类玩家的定位分析

### 运营现有厂商和 SORS（记录系统）
**被隔离且优先考虑当前状态。** Salesforce、ServiceNow 和 Workday 都在其现有平台上构建 Agent，但这些 Agent 继承了底层架构。Salesforce 基于当前状态存储构建：它知道机会现在是什么样子，而不是决策做出时的样子。当折扣获得批准时，证明它合理的上下文没有被保留。你无法重放决策时的世界状态，这意味着你无法审计决策、从中学习或将其用作先例。

更重要的是，没有现有厂商能看到跨系统图景。支持升级依赖于 CRM 中的客户层级、账单中的 SLA 条款、PagerDuty 中的最近中断、以及标记流失风险的 Slack 线程。没有一个单一现有厂商处于该路径中。

### 数据仓库玩家
**完全在读取路径，而非写入路径。** Snowflake 和 Databricks 将自己定位为智能层，但仓库通过 ETL 在决策做出后接收数据。它们接收决策的**输出**，而非产生决策的推理。接近 Agent 构建的地方与处于决策发生的执行路径中是不同的。

### 系统-Agent 初创公司
**具有结构性优势**，因为它们默认处于写入路径。当 Agent 分类升级、响应事件或决定折扣时，它从多个系统拉取上下文、评估规则、解决冲突并行动。因为它执行工作流程，所以在决策变得具有约束力的时刻捕获理由——不是事后通过 ETL，而是在当下，作为一等记录。

由于企业决策追踪对于普通访问控制过于敏感，构建这一层的公司必须支持**许可推理 (permissioned inference)**，而不仅仅是许可检索。律师事务所不能让一个客户的先例悄悄地影响竞争对手的推理；医疗保健组织不能允许运营历史通过抽象泄露。解决这个问题的公司赢得的信任与数据一样确定地复合。

## 平台与应用机会

这既是平台机会，也是应用机会。每个现有厂商已经在盯着这个奖品，这告诉你它有多大。

### 对于某些公司
策略是拥有自己的上下文图谱——构建专有的决策基础设施，使其 Agent 在其领域内比任何其他公司的更智能。对于这些公司，将出现一个新的基础设施层：在企业规模上构建、保护和查询上下文图谱的"镐和铲"。

### 对于应用公司
机会是构建上下文图谱本身——特定领域的、跨功能的、复合的决策记录系统。我们看到这些沿三个轴出现：

1. **运营上下文图谱**（公司如何战术性运行）
2. **面向客户的上下文图谱**（公司如何销售、支持和保留——销售、承保、账户管理）
3. **战略上下文图谱**（高管如何做出战略决策）

每个都是具有不同机密性要求、不同决策模式和不同结果信号的独立表面。

## 从检索到预测的转变

一旦这些图谱变得足够密集，游戏就从检索变为预测。你可以问的不是"我们上次如何处理这个？"而是"如果我们以这种方式构建交易，可能会发生什么？"——不是基于通用训练数据，而是基于你组织的实际决策历史和结果。这种能力尚未完全存在，但组装这些基础的公司正在朝这个方向构建。

## 每个企业垂直领域的机会

每个企业垂直领域——法律、保险、医疗、金融服务、采购、安全——都有数十年来积累的**机构判断 (institutional judgment)**，这些判断从未被结构化、从未被复合、从未被操作化。

这就是让 2000 美元/小时的合伙人值 2000 美元/小时的原因。前沿模型正在提高下限，但它们没有提高上限。

**上限是机构性的。** 它是积累的、特定领域的、经过结果测试的推理——关于这个组织在这些约束下如何做出决策。这才是复合的，也是更好的基础模型无法复制的。这也是最终变得可捕获、可结构化、可学习的。

## 结论

消费巨头通过复合行为追踪建立了万亿美元帝国。**企业等价物现在才成为可能，而且奖品可能更大。** 构建使这成为现实的基础设施的公司将定义企业价值的下一个时代。
